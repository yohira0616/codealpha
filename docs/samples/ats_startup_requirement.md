# 【架空企業のテスト用要件】採用管理システムのご相談(株式会社ブルーポンド)

> ※このドキュメントは見積もりシステムのテスト用に作成した架空の要件テキストです。登場する企業・人物・数値はすべて架空のものです。

---

はじめまして。株式会社ブルーポンドで人事をやっている高橋です。カジュアル面談でお話しした採用管理システムの件、要望をまとめました。

## 会社について

BtoB SaaS(在庫管理系)を作っている創業5年目のベンチャーです。社員は35名、うちエンジニアが20名。今年から採用を強化していて、年間で30名くらい採る計画です。エンジニア採用が中心ですが、セールスやCSも採ります。

今の採用管理はスプレッドシート+Slack+Googleカレンダーの合わせ技で、正直破綻しかけています。候補者が今どの選考ステップにいるのか、誰がボールを持っているのかが分からなくなって、返信が3日遅れて辞退された、みたいな事故が月1くらいで起きています。

## 予算と時期について(重要)

**予算は上限500万円です。**これを超える提案は社内を通せません。正直、Herp とか HRMOS みたいな SaaS も検討したのですが、後述のエンジニア採用特有のワークフロー(技術課題まわり)が合わなくて自作に傾いています。予算内に収まるよう、機能の優先順位付けや「ここは初期リリースから落とす」という提案も歓迎します。

時期は、来期の採用ピーク(4月)に間に合わせたいので、できれば3ヶ月くらいで初期版が動くと嬉しいです。

## やりたいこと

### 候補者管理(これが本丸)

- 候補者の一覧と詳細。今どの選考ステップにいるか(書類選考→カジュアル面談→技術課題→1次面接→最終面接→オファー→内定承諾)をカンバンっぽく見たい
- ステップごとの滞留日数が見えて、「3日以上動いていない候補者」が一目で分かるようにしてほしい。これが一番の事故対策です
- 候補者ごとの対応履歴(誰がいつ連絡したか、面談メモ)
- 応募経路の記録(媒体、エージェント、リファラル、直接応募)。あとで経路別の歩留まりを見たい

### 求人管理

- 職種ごとの求人票の管理(公開中/停止中)。求人票は今Notionに書いているので、最悪リンク管理でもいいです

### 面接調整まわり

- 面接官のアサインと日程調整。Googleカレンダーと連携して面接官の空き時間から候補日を出せると最高ですが、難しければアサインと予定の記録だけでもいいです
- 面接が決まったらSlackの採用チャンネルに通知してほしい(これは必須に近い)

### エンジニア採用特有のワークフロー

- 技術課題(GitHubリポジトリのURL)の送付記録と、提出期限・提出状況の管理
- 課題レビューの担当割当と、レビュー結果(合否+コメント)の記録。レビュアーはエンジニアなので、Slack通知で完結できるとありがたい

### 評価・選考

- 面接ごとの評価フォーム(スコア+コメント)。面接官が他の人の評価を見る前に自分の評価を出す方式にしたい(バイアス防止)
- 最終的な合否判断の記録

### レポート

- 採用ファネル(応募→各ステップの通過率→内定承諾率)を職種別・経路別に見たい。四半期の経営会議で使います。最初はCSV出力できれば、グラフはスプレッドシートでやります

## その他

- 使うのは人事2名+面接官(エンジニア中心に15名くらい)+経営陣。権限は「人事(全部見える)」「面接官(自分の担当だけ)」「閲覧(経営陣)」の3種類くらい
- 候補者の個人情報を扱うので、アクセス権限はちゃんとしたいです。不採用者のデータは一定期間後に消す運用にしたい(法律まわりは詳しくないので相談させてください)
- スマホは面接官が評価を出すときに使うくらいなので、レスポンシブで見られれば十分です
- 応募の受付そのもの(応募フォーム)は各媒体でやるので、今回のシステムには要りません。媒体からの取り込みは、当面手入力で我慢します
